Which authentication method to choose for the project?

Authentication is common way to handle security for all applications. This is only way to answer the question “who you are?” to the application, when comes to stateless architecture or service oriented architecture we got lot of new concepts and technologies in the market. You can find details of different type of methods in the internet. In this article, I wrote about how to choose the authentication method.

Types of Authentication

  • Cookie-Based authentication
  • Token-Based authentication
  • OpenId
  • SAML
  • Third party access

Stateless authentication is used by all around the world. Above are the common practicies for web or mobile authentication. I can say that no one is perfect for your project. But, you have to choose best suitable method for your project.

SSO(Single Sign ON)

Nowadays, SSO(Single Sign ON) is used by big tech companies. If you have multiple portals for different users, you should definately go for the SSO. As a user, he/she didn't have to remember different passwords. Also, they don't have to login multiple times. You can choose different service provider for your project like: Amazon AWS, Microsoft Azure etc.

Open Authentication(OAuth)

If you want to integrate third party authentication like Login via Facebook or Google or Linkedin or Github or any other specific site, you have to go for Open Authentication(OAuth). OAuth is a protocol that allows an application to authenticate against server as a user. The recommendation is to implement OAuth 1.0a or OAuth 2.0. OAuth 2.0 relies on HTTPS for security and it currently implemented by Google, Facebook, Twitter etc.

Token-Based authentication

If you have only one portal, you can opt for the token base authentication. JSON web tokens are mostly used. You can easily find the library or package in any popular programming language. So, you don't have to write code for generating or verifying the tokens. I'm gonna use this method. I found the popular library for Python Flask Project: Flask JWT Extended
