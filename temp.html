Understanding AsyncStorage, Redux: Why and How it can be used in React-Native?

Why Redux?

First, we have to understand why redux is needed in our application. Recently, I completed mobile onboarding application. After that, user moved to the homepage. However, when user refreshed the application; mobile onboarding screens again shown to the user. The purpose of mobile onboarding screen to show main features of the application to the user---This should be one time only--.

AsyncStorage

As you know, react native manages state data component wise and you need to store boolean flag (ex. userOnboardingCompleted?) to create condition on mobile onboarding screens. So, I looked for the local storage and found well known library "AsyncStorage" for react-native. This thing worked for me. Do not use react-native core component for that as it is deprecated in new versions. Follow this tutorial to start with AsyncStorage and trust me it is very easy part.

Redux

Now, I want to use apiToken in every component. You might be thinking: it is easy to get values from the local storage using AsyncStorage getItem function. But, it is bad designing because everytime your application fetch value from the storage so it takes much time to render new pages.

This time redux come into the picture. Redux is a state manager. In other way, it is state management libraries beautifully keep logic and behaviours abstracted away from the UI. So, your application reads local storage data one time and stores it into the memory using state. Now, redux passes this data to every component so it will save lot of time.

Challenges

Learning redux is quite difficult as you have to learn many new things. When I approached Redux for the first time today I was confused. There is too much terminology: action, reducer, middleware. But on top of that it wasn't clear to me how were all the moving parts glued together? There were some minions or what? In Redux there are no minions (unfortunately).
