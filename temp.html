Start the emulator from the command line in MAC OS

I'm developing mobile application in react-native. Manytimes I want to test my application in android device. It's very hard to test application by building apk and run on physical device. If you are developing application in react-native or any cross platform, you might want to run application on iOS as well android device.

It's easy to test application in ios emulator because react-native run-ios can directly opens the emulator in MAC OS. However, react-native run-android command can't open android emulator. You have to open android studio to start the emulator.

Solution

You can start the emulator using command line. Firsly, your system have installed android studio. Then, go to the android sdk folder. Default installation path in MAC os is: 

        
            /Users/yoursystemname/Library/Android/sdk/emulator

Change your system name in above path. After that, 

        
            emulator -list-avds
OR 
        
            ./emulator -list-avds

Then copy the avd_name that you want to start. After that run following command: 

        
            emulator @avd_name
OR 
        
            ./emulator @avd_name

Now, your emulator will start. Many people find this problem:"PANIC: Missing emulator engine program for 'x86' CPUS." In that case, you just have to go to the emulator directory instead of toools. If you face any problem then check above screenshot for more details.
