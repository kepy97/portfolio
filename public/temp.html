Building web application from scratch

I have started working on creating a web application at ABB. I'll be working on this project alone so I have to gather all the requirement and decide which programming languages to use to full-fill the requirement. I think spending time on project plan instead of directly jump on to the programming is less burden and beneficial decision later on.

Language Selection

When a programming language decision must be made, programs should conduct a trade study to determine the best computer programming language, or mix of programming languages, to be used to satisfy system life cycle requirements. Each programming language has their own benefits but choosing the right programming language saves a lot of time. How?

  • Easiness of writing the code
  • Faster performance
  • Reuse --- You can use existing library/packages to complete task
  • Support --- If technology is well known so you can get lot of support and bug solution from the internet
  • Extent of compatibility with and impact of other related direction

To make a data-centric web app from the bottom-up, it is advantageous to plan below things:

  • Web front end (HTML, CSS, Javascript) - for the look and feel of your web app
  • Backend language (e.g. Node.js, Python, Ruby) - control how your web app works
  • Database (Oracle, PostgreSQL) - to store the data of the application
  • DevOps (Github, Jenkins) - Deploying / hosting your web app

Frontend technology

User experience is the number one priority for every business in the market today. As a developer, even the brief you get on your project talks about the simplicity of the user interface it is supposed to feature. No matter how complex functions and operations happen in the background, what users see and experience has to be seamless.

Backend language

The backend programming language is also known as server side programming. The name given to all types of programs which run on the web server. They process the user input, interact with the databases and control what content is served back to the client as a response to his request. It is written in a number of programming languages including PHP, NodeJS, Python, etc and has full access to the server’s OS and the programmer can chose the language he/she wants to code in.

I'm going to code in the Node.js language. Node.js is the newest in the list (released in 2009) and the quickest growing today. It provides the ability to run JavaScript code server-side. The great thing is that I do not have to learn a new language for back-end development. I'll use Javascript as backend and frontend development. It's also helpful to full stack development.

Database

There are hundreds of tech-heavy database reviews out there, but they don’t always give clear guidance on the first step in selecting a database: choosing the best general type for a specific application. All databases are not created equal. Each has specific strengths and weaknesses.

I choose PostgreSQL database to store the data for our application. I have an option to choose NoSQL database but our application have certain limits so I went for RDMS. Also, postgre is an Enterprise-class relational database system. It is easy to setup and installs. It offers support for SQL and NoSQL. It has a great community which happy to serve you when you are facing issues while using PostgreSQL.
