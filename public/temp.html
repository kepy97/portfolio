Performance Improvement in Project

Recently, I got some time from the developing the project. So, I thought let's take a look to improve the performance of the project.

How to improve?

There are several ways improve the performance. Like? You can start first from cleaning up code, removing unnecessary imports in the file, deleting not needed files. Also, you can reorganize your project structure by dividing into multiple components.

Above are the very basic things that you can do in your project. After that, you can take a look at your application main section which is used many times when user open the application. This is the section which executes several times. It can be homepage of your application. If there's multiple loops are running, try to remove that first.

Loops?

Yes, loops like for, while are downgrade your application performance. You can move that loop code from backend to front-end or get specific formatted data from the database. Always, try to avoid using loops in the backend except there's not any other way.

What next?

By following above steps, your application will be significantly improved. Now, you can focus on database part.

In software application, there are major performance based on backend, and it continuously interacts with the database. You must be using ORM in backend to get the data. You can find many guidelines about how to improve performace in your ORM library. I'm using SQLAlchemy in python flask framework.

I found pretty good article about Master SQLAlchemy Relationships in a Performance Friendly Way. If you can't find anything there are some basic rules that you can avoid like:

  • Retrieving an entire result set when only the count is needed
  • Retrieving entire models when only a few columns are needed
  • Updating one object at a time inside a loop
  • Triggering cascading deletes
  • Using the ORM for a bulk record import
  • Calling commit or flush multiple times

I've followed above all rules in my project. Now, performance is drastically changed. I hope this list of common pitfalls can help you avoid these issues and keep your application running smoothly. As always, when diagnosing a performance problem, measurement is key. Most databases offer performance diagnostics that can help you pinpoint issues, such as the PostgreSQL pg_stat_statements module.
